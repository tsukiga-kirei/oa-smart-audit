# 仪表盘功能规范 (Dashboard Feature Specification)

## 概述

仪表盘（Overview）是 OA 智审系统所有用户的主要登录入口页面（`/overview`）。其主要目的是根据用户的角色和权限提供相关的统计组件（Widgets）。仪表盘支持高度的用户自定义，同时强制执行对受限组件的安全访问控制。

---

## 基于角色的显示规则

组件的加载基于从当前激活角色（`business`, `tenant_admin`, `system_admin`）派生的 `userPermissions` 进行过滤。

### 1. 业务人员 (Business User)
侧重于日常的审核工作流、定时任务和流程反馈。

*   **审核概览**: 取代了原来的“今日审核概览”，重点展示累计总数、已通过、已驳回和**已归档**的数量。
*   **待办任务**: 显示当前需要处理的流程数量。
*   **审核趋势**: 审核流程数量的每周柱状图。
*   **定时任务**: 最近或在运行的定时任务，指示后台自动化审核的情况。
*   **归档复盘**: 有关近期归档复盘日志。
*   **最新动态**: 常规全系统或用户事件。

*(已移除：部门分布、AI 模型表现)*

### 2. 租户管理员 (Tenant Administrator)
监控其特定租户层级内的整体使用情况。

*   **部门分布使用情况**: 显示租户内不同部门的总审核量（原为业务人员组件，已重命名）。
*   **最近动态**: 租户内的系统级事件。
*   **AI 模型表现**: 租户内整体 Token 用量、AI 响应时间和准确率。
*   **租户资源用量**: 存储和 Token 的配额及消耗情况追踪。
*   **用户活动排行**: 租户内最活跃的用户排行。

*(已移除：规则覆盖率)*

### 3. 系统管理员 (System Administrator)
全局监督平台基础设施和系统运行指标。

*   **全局监控**: 显示集群服务运行情况、降级状态、CPU 和内存等指标。该功能已整合至 Dashboard，原独立页面已移除。
*   **租户概览**: 列出活跃和已停用的租户。
*   **API 调用指标**: 平台接口的延迟、调用量和成功率监控。
*   **运行指标**: 系统关键运行指标概览，包括 API 成功率、模型响应时间、延迟分布等。
*   **最近告警**: 系统告警与异常事件记录。

## 布局结构

## 布局结构

布局使用 12 列的 CSS 网格 (`widget-grid`)，利用原生的 `flex`、`grid` 及 `order` 属性结合 HTML5 的拖拽 API，以实现性能开销最小的动态全布局特性。

组件共分为三种默认尺寸：
*   `size: 'sm'`: 小卡片，占据 4 列 (`widget--sm`)。
*   `size: 'md'`: 中卡片，占据 6 列 (`widget--md`)，大量使用以避免过多的留白，也是默认的首选尺寸。
*   `size: 'lg'`: 大卡片，横跨所有 12 列 (`widget--lg`)。

## 自定义与个人设置

为了最大限度地提升操作体验及降低冗余视图，个人仪表盘的排版设置直接通过仪表盘前端页面 (`/overview`) 右上角的“自定义仪表盘”进行管理。

在自定义模式 (`customizing` 模式) 下：
1. **内容控制**：顶部面板会自动列出属于当前访问者职权的**所有可用卡片**。用户只需点击隐藏或显示眼睛图标，即可决定要保留的监控信息，所有更改均实时反馈到底层网格之中。
2. **尺寸管理**：点击对应图表卡片右上角弹出的重置尺寸(`AppstoreOutlined`)按钮，可以原位、循环对图表卡片的大小（`sm`/`md`/`lg`）进行覆盖修正以更好地填补页面留空区域。
3. **定位排序**：通过在虚线编辑状态下直接用鼠标左键按住拖挪当前面板的任何区域并将它丢弃在其它面板框内，底层布局的 `order` 顺序便会自动刷新以执行重排。

点击保存后，所有的设置将基于 **【当前用户的当前切换身份】**，组合成为包含 `enabled_widgets` 和 `widget_sizes` 在内的视图首选项并实现持久化。这使得系统拥有记忆用户对不同页面的关注焦点的能力。

## API 集成概览

*   **GET /api/user/dashboard-prefs**: 获取当前用户的偏好配置（包括 `enabled_widgets` 数组与修改过的尺寸对象 `widget_sizes`）。
*   **PUT /api/user/dashboard-prefs**: 覆盖保存用户的自定义配置，后端将静默丢弃其中任意非法或提权操作的组件数据。
*   **GET /api/audit/history**: 此接口现在直接为仪表盘的数据统计提供支持，特别是总览卡片中的 `archived`（已归档）数值。
