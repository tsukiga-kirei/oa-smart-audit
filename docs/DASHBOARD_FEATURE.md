# 仪表盘功能规范 (Dashboard Feature Specification)

## 概述

仪表盘（Overview）是 OA 智审系统所有用户的主要登录入口页面（`/overview`）。其主要目的是根据用户的角色和权限提供相关的统计组件（Widgets）。仪表盘支持高度的用户自定义，同时强制执行对受限组件的安全访问控制。

---

## 基于角色的显示规则

组件的加载基于从当前激活角色（`business`, `tenant_admin`, `system_admin`）派生的 `userPermissions` 进行过滤。

### 1. 业务人员 (Business User)
侧重于日常的审核工作流、定时任务和流程反馈。

*   **审核概览**: 取代了原来的“今日审核概览”，重点展示累计总数、已通过、已驳回和**已归档**的数量。
*   **待办任务**: 显示当前需要处理的流程数量。
*   **审核趋势**: 审核流程数量的每周柱状图。
*   **定时任务**: 最近或在运行的定时任务，指示后台自动化审核的情况。
*   **归档复盘**: 有关近期归档复盘日志。
*   **最新动态**: 常规全系统或用户事件。

*(已移除：部门分布、AI 模型表现)*

### 2. 租户管理员 (Tenant Administrator)
监控其特定租户层级内的整体使用情况。

*   **部门分布使用情况**: 显示租户内不同部门的总审核量（原为业务人员组件，已重命名）。
*   **最近动态**: 租户内的系统级事件。
*   **AI 模型表现**: 租户内整体 Token 用量、AI 响应时间和准确率。
*   **租户资源用量**: 存储和 Token 的配额及消耗情况追踪。
*   **用户活动排行**: 租户内最活跃的用户排行。

*(已移除：规则覆盖率)*

### 3. 系统管理员 (System Administrator)
全局监督平台基础设施和系统运行指标。

*   **全局监控**: 显示集群服务运行情况、降级状态、CPU 和内存等指标。这完全替代了旧版的 `/admin/system/monitor` 独立页面功能。
*   **租户概览**: 列出活跃和已停用的租户。
*   **API 调用指标**: 平台接口的延迟、调用量和成功率监控。

---

## 布局结构

布局使用 12 列的响应式 CSS 网格 (`widget-grid`)。

*   `widget--sm`: 小卡片，占据 4 列。
*   `widget--md`: 中卡片，占据 6 列（大量使用以避免过多的留白）。
*   `widget--lg`: 大卡片，横跨所有 12 列。

## 自定义与个人设置

为了减少冗余，个人的仪表盘配置现在直接通过仪表盘页面 (`/overview`) 上的“自定义仪表盘”面板进行管理（`customizing` 模式）。个人设置页面 (`/settings`) 中的独立的“仪表盘”标签页已被彻底移除。用户可以主动显示或隐藏允许访问的组件，所有的更改将自动通过 Mock API 包装器进行持久化保存（后续将对接真实后端 API）。

---

## API 集成概览

*   **GET /api/user/dashboard-prefs**: 获取 `enabled_widgets` 数组。
*   **PUT /api/user/dashboard-prefs**: 保存用户的自定义配置。后端将执行安全检查，并在保存时静默丢弃任何无权访问的 Widget ID。
*   **GET /api/audit/history**: 此接口现在直接为仪表盘的数据统计提供支持，特别是总览卡片中的 `archived`（已归档）数值。
