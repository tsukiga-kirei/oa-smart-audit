## 4. 用户反馈问题分析

### 4.1 "没到 7 天就过期了"

**可能原因**:
1. 在其他设备登出导致 Refresh Token 被吊销
2. 用户本地时间不准确
3. 使用隐私模式浏览器

**排查建议**:
- 检查 Redis 中是否存在对应的黑名单记录
- 检查用户本地时间与服务器时间差异

### 4.2 "到期后没有自动退出"

**根本原因**:
- 前端只在路由切换或 API 请求时检查 Token 有效性
- 用户停留在同一页面不会触发检查

**修复方案**:
- 添加定时器定期检查 Token 有效期
- 监听页面可见性变化事件

## 5. 代码质量亮点

### ✅ 优秀实践

1. **完善的黑名单机制**
   - 登出时将 Token JTI 加入 Redis 黑名单
   - 防止已登出 Token 被滥用

2. **并发刷新保护**
   - 前端使用 `isRefreshing` 标志和订阅队列
   - 防止多个请求同时触发刷新

3. **灵活的规则系统**
   - 支持 mandatory/default_on/default_off/custom 四种作用域
   - mandatory 规则不可被用户覆盖

4. **脏数据自动清理**
   - 租户删除规则后，用户覆盖自动过滤
   - 避免产生孤立数据

5. **完善的权限控制**
   - 用户权限锁定机制（AllowCustomFields/Rules/Strictness）
   - 访问控制白名单机制
